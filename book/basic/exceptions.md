## Исключения
Проще говоря обработчики ошибок.Это способ `ловить` ошибки в runtime 
и правильно их обрабатывать.Чтобы ОС не вмешивалась, и могли хоть как-то спасти
программу. 

Есть три основные команды:
* `try/catch`: Конструкторы чтобы ловить ошибки.
```
try{
    ... // опасный код 
} catch( опредленная ошибка ) {
    ... // Тут реакция на эту ошибку

} catch(...){ // `...` - значит обработка всех остальных ошибок 
    ... // 
}

```

В коде может быть сколько угодно `catch`. После одного `try`


* `throw`: Чтобы `бросить` ошибку на которую мы сможем нормально среагировать.

```
    if ...;
        ... 

    // Если наши условия не совпали или что-то не так 
    throw "Error!!!";
```

Позже сообщение переданное в `throw` ("Error!!!") мы если что можем поймать в `catch`

```
try{
    ... // Опасный код
} catch (const char* message) {
    cout << message << endl;
}
```



## Типизация
Обычно в коде чтобы реагировать на разные ошибки по разному используются разные классы

```
try{

}catch(const DatabaseError& e){

}catch(const SomeException& e){

}catch(const exception& e){

}

```

Можно создавать свои кастомные классы. 

```
class CustomException{
    ...
};
```
И после вписывать туда что угодно, главное написать throw и передать туда обьект созданный
с помошью этого класса 
```
throw CustomException{...};
```


Все теперь можно ловить

```

catch(const CustomException& e){
    ...
}
```

`catch` может распознавать обьекты и их производные классы
Если у тебя есть один общий класс-родитель

```
class ParentExcp{};

```

и от него отходят производные классы
```

class Child1 : public ParentExcp{};

class Child2 : public ParentExcp{};

```

то ошибки вызванные путем `Child1` и `Child2` могут быть пойманы
```
catch(const ParentExcp& e){
    ... 
}
```

И в любом `catch` если ловите по обьекту обязательно используем ссылки 
иначе пройзодет обрезка обьекта.




## Noexcept 

`noexcept` - модификатор который обозначает что функция нивкоем случае не выкинет исключение.

```
void func() noexcept {
    ... 
}
```

Он нужен для оптимизации, дело в том что если функция может выбросить исключение то компилятор 
добавляет дополнительный невидимый функционал и это занимает память и время, однако с `noexcept`
вы говорите компилятору что эту функцию можно не усложнять она не вызовет ошибок.
Но если ошибка все же будет,то вся программа рухнет.
